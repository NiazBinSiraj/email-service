Success Response (200)

{
  "status": "success",
  "message": "Email sent successfully",
  "data": {
    "messageId": "<unique-message-id@gmail.com>",
    "recipients": 1,
    "subject": "Test Email",
    "sentAt": "2025-08-03T10:30:00.000Z"
  }
}

Error Response (400)

{
  "status": "error",
  "message": "Validation failed",
  "errors": [
    {
      "field": "to",
      "message": "Recipient email address is required",
      "value": ""
    }
  ]
}

Validation Rules

to Required, valid email(s), max 50 recipients
subject Required, 1-200 characters
message Required, 1-10,000 characters
cc Optional, valid email(s), max 20 recipients
bcc Optional, valid email(s), max 20 recipients

Rate Limiting

The API implements rate limiting to prevent abuse:

  • Window: 15 minutes (configurable)
  • Max Requests: 100 per window (configurable)
  • Response: 429 status with retry information

Getting Started

Get your email service up and running in minutes

1

Clone & Install

git clone https://github.com/NiazBinSiraj/email-service.git
cd email-service
npm install
2

Configure Environment

copy .env.example .env

Edit .env with your Gmail credentials:

GMAIL_USER=your-email@gmail.com
GMAIL_PASS=your-app-password
PORT=3000
NODE_ENV=production
3

Start the Server

npm start

Expected output:

‚úÖ Email service is running on port 3000
üåç Environment: production
üìß Gmail user configured: Yes
4

Test the API

Run the included test script:

.\test-api.ps1

Or test manually:

curl http://localhost:3000/health

üìã Important: Gmail App Password

You must use a Gmail App Password, not your regular password:

  1. Enable 2-Factor Authentication on your Gmail account
  2. Go to Google Account Security
  3. Navigate to "2-Step Verification" ‚Üí "App passwords"
  4. Generate a new app password for "Mail"
  5. Use this 16-character password in your .env file
